# Spacing System

## Проблема
Везде используются хардкод значения spacing без единой системы:
- `p-4`, `p-6`, `p-8` - разные значения для одинаковых случаев
- `gap-3`, `gap-4`, `gap-6` - нет консистентности
- `space-y-1`, `space-y-8` - огромная разница без логики
- `mb-2`, `mb-4` - случайные значения

## Решение: Единая система spacing

### Принципы
1. **Консистентность**: Одинаковые случаи = одинаковые значения
2. **Иерархия**: Четкая система увеличения отступов
3. **Семантика**: Значения имеют смысл

### Система значений

**Базовые отступы (для внутренних элементов):**
- `p-4` (16px) - минимальный padding для карточек/контейнеров
- `p-6` (24px) - стандартный padding для карточек
- `p-8` (32px) - большой padding для основных секций

**Вертикальные отступы между элементами:**
- `space-y-2` (8px) - очень маленький (для плотных списков)
- `space-y-4` (16px) - маленький (для форм внутри карточек)
- `space-y-6` (24px) - стандартный (для элементов в формах)
- `space-y-8` (32px) - большой (для секций на странице)

**Горизонтальные отступы:**
- `gap-2` (8px) - очень маленький
- `gap-4` (16px) - стандартный для элементов в ряду
- `gap-6` (24px) - большой для групп элементов

**Margin для заголовков:**
- `mb-2` (8px) - маленький отступ после заголовка
- `mb-4` (16px) - стандартный отступ после заголовка
- `mb-6` (24px) - большой отступ после заголовка

### Применение

**Card компоненты:**
- CardHeader: `px-8 pt-8 pb-6`
- CardBody: `px-8 pb-8`
- CardFooter: `px-8 pb-8 pt-6`

**Формы:**
- Форма: `space-y-6` (между полями)
- Input wrapper: `space-y-3` (между label и input)

**Списки:**
- Список элементов: `space-y-4` или `gap-4`
- Секции на странице: `space-y-8`

**Layout:**
- Main content: `p-6 md:p-8` (responsive)
- Sidebar navigation: `p-6 space-y-2`
