# Architecture Agent

## Описание

Architecture Agent - это мета-агент для Cursor, который проектирует системную архитектуру, схему базы данных и API контракты на основе продуктовых требований.

## Назначение

Architecture Agent создает архитектурную документацию для CRM системы, включая:
- Системную архитектуру (компоненты, взаимодействия)
- Схему базы данных (таблицы, связи, индексы, RLS)
- API контракты (REST endpoints, схемы запросов/ответов)
- Архитектуру интеграций (WhatsApp, AI, переводчик)

## Использование

Когда нужно спроектировать архитектуру системы, используй инструкции из:
- `docs/agents/meta-agents/architecture/instructions.md`
- `docs/agent-templates/architecture/` (шаблоны для архитектурных документов)

## Входные данные

Для работы Architecture Agent нужны:
- `docs/product/requirements.md` - требования к продукту
- `docs/product/user-stories.md` - user stories
- `docs/product/business-processes.md` - бизнес-процессы
- `PROJECT_PLAN.md` - технологический стек
- `CONTEXT.md` - контекст проекта

## Выходные артефакты

Architecture Agent создает:
- `docs/architecture/system-design.md` - системная архитектура
- `docs/architecture/database-schema.md` - схема базы данных
- `docs/architecture/api-contracts.md` - API контракты
- `docs/architecture/integrations.md` - архитектура интеграций (опционально)

## Примеры использования

### Пример 1: Проектирование системной архитектуры

**Запрос**: "Спроектируй системную архитектуру для мультитенантной CRM"

**Результат**: Создан `docs/architecture/system-design.md` с описанием всех компонентов системы, их взаимодействий и принципов мультитенантности.

### Пример 2: Проектирование схемы базы данных

**Запрос**: "Спроектируй схему БД с поддержкой мультитенантности"

**Результат**: Создан `docs/architecture/database-schema.md` с описанием всех таблиц, связей, индексов и RLS политик.

## Ограничения

- Архитектура должна быть реализуема с технологическим стеком из `PROJECT_PLAN.md`
- Мультитенантность должна быть заложена с самого начала
- Архитектура должна поддерживать масштабирование на множество организаций
