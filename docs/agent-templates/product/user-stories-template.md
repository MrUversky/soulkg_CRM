## Заполнение

1. **Эпики**: Группируй связанные stories по функциональным областям
2. **User Stories**: Используй формат "Как [роль], Я хочу [действие], Чтобы [цель]"
3. **Критерии приемки**: Конкретные, измеримые критерии успеха
4. **Приоритет**: Используй информацию из `docs/product/organization-info.md` (раздел 14 - приоритеты)
5. **Зависимости**: Указывай зависимости между stories

## Примеры

### Пример User Story для менеджера

```markdown
#### US-001: Просмотр списка клиентов
**Как** менеджер продаж,  
**Я хочу** видеть список всех клиентов с их статусами,  
**Чтобы** быстро находить нужных клиентов и понимать текущее состояние воронки.

**Критерии приемки**:
- Отображается список всех клиентов организации
- Для каждого клиента показывается: имя, телефон, текущий статус, дата последнего контакта
- Список можно фильтровать по статусу
- Список можно сортировать по дате последнего контакта
- При клике на клиента открывается его профиль

**Приоритет**: Must have  
**Оценка**: 3 story points
```

### Пример User Story для AI-агента

```markdown
#### US-010: Квалификация лида
**Как** AI-агент квалификации,  
**Я хочу** задавать вопросы лиду для определения его потребностей,  
**Чтобы** автоматически квалифицировать лида и обновить его статус на `qualified`.

**Критерии приемки**:
- AI-агент задает вопросы из анкеты квалификации (даты, количество человек, уровень активности, бюджет)
- Ответы сохраняются в профиле клиента
- При успешной квалификации статус автоматически обновляется на `qualified`
- Менеджер получает уведомление о квалифицированном лиде

**Приоритет**: Must have  
**Оценка**: 5 story points
```

### Пример User Story для клиента

```markdown
#### US-020: Получение ответа на вопрос
**Как** клиент,  
**Я хочу** получить быстрый ответ на мой вопрос через WhatsApp,  
**Чтобы** понять детали тура и принять решение.

**Критерии приемки**:
- Клиент отправляет сообщение в WhatsApp
- Система получает сообщение и обрабатывает его через AI-агента
- Клиент получает ответ в течение 5 секунд
- Ответ соответствует тону и стилю общения Soul KG

**Приоритет**: Must have  
**Оценка**: 8 story points
```

## Правила написания User Stories

1. **Одна story = одна функция**: Не объединяй несколько функций в одну story
2. **Конкретность**: Избегай общих фраз, будь конкретным
3. **Ценность**: Каждая story должна приносить ценность пользователю
4. **Измеримость**: Критерии приемки должны быть измеримыми
5. **Независимость**: Stories должны быть независимыми (насколько возможно)
