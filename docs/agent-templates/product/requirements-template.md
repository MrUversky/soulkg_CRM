2. **Функциональные требования**: Используй информацию из `docs/product/organization-info.md` (раздел 14 - приоритеты)
3. **Нефункциональные требования**: Используй информацию из `PROJECT_PLAN.md` (технологический стек)
4. **Ограничения**: Используй информацию из `docs/product/organization-info.md` (раздел 12) и `CONTEXT.md`
5. **Интеграции**: Используй информацию из `docs/product/organization-info.md` (раздел 14 - каналы связи)

## Примеры

### Пример функционального требования

```markdown
### 2.1 Управление клиентами

**FR-001: Создание клиента**
- Система должна позволять создавать нового клиента с полями: имя, телефон, email
- Клиент автоматически получает статус `new_lead`
- Клиент привязывается к организации (tenant)

**FR-002: Просмотр истории общения**
- Система должна отображать всю историю общения с клиентом
- История должна включать: дату, время, отправителя (AI/человек), текст сообщения
- История должна быть отсортирована по дате (новые сверху)
```

### Пример нефункционального требования

```markdown
### 3.1 Производительность

**NFR-001: Время отклика**
- Система должна обрабатывать до 20 клиентов одновременно
- Время отклика на действия пользователя не должно превышать 2 секунд
- Время обработки сообщения WhatsApp не должно превышать 5 секунд
```
