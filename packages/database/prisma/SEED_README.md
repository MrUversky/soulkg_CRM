# Seed Data для базы данных

## Описание

Seed скрипт создает начальные данные для разработки и тестирования.

## Что создается

### 1. Тестовая организация
- **Название**: Soul KG
- **Slug**: `soul-kg`
- Автоматически создается при первом запуске seed

### 2. STATUS_DETECTION Agent Configuration
- **Тип**: `STATUS_DETECTION`
- **Название**: `default`
- **Промпт**: Детальный промпт для определения статуса клиента на основе диалога
- **Настройки**:
  - Model: `openrouter/gpt-4o-mini`
  - Temperature: `0.3`
  - Max Tokens: `100`
  - Timeout: `30000ms`
  - Retry Attempts: `3`

## Использование

```bash
# Запуск seed
npm run db:seed

# Или через Prisma CLI
npx prisma db seed
```

## Повторный запуск

Seed скрипт использует `upsert`, поэтому безопасно запускать его несколько раз:
- Если организация существует - используется существующая
- Если промпт существует - обновляется
- Если не существует - создается новый

## Расширение seed данных

Для добавления новых seed данных отредактируйте `prisma/seed.ts`:
- Добавьте создание пользователей
- Добавьте создание клиентов
- Добавьте создание продуктов
- И т.д.

См. `docs/development/SEED_DATA.md` для полного плана seed данных.
